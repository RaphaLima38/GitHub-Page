# Versionamento do repositorio

## Estrutura adotada

O repositorio foi organizado com:

- site do portfolio na raiz
- projetos separados em subpastas
- documentacao separada em pasta propria

## Boas praticas utilizadas

- commits com objetivo claro
- branch principal `main`
- branches de funcionalidade para mudancas maiores
- push para o GitHub apos alteracoes significativas
- documentacao em README para explicar cada projeto

## Fluxo usado

1. Planejar a mudanca
2. Criar ou usar uma branch de funcionalidade
3. Implementar a alteracao
4. Commitar
5. Enviar para o GitHub
6. Abrir pull request quando necessario

## Motivo dessa organizacao

Essa estrutura facilita:

- leitura do repositorio
- manutencao
- reaproveitamento de projetos
- apresentacao profissional no GitHub
