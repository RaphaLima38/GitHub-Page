# Segurança e Controle de Acesso

## Situação do projeto

O PortfolioHUB é um site estático, sem banco de dados, sem login próprio e sem coleta de dados de visitantes. Por isso, a parte de segurança se concentra principalmente no repositório, na publicação e no cuidado com informações expostas.

## Políticas adotadas

1. Não publicar senhas, tokens, chaves de API ou dados pessoais sensíveis.
2. Manter no repositório apenas arquivos necessários para a entrega.
3. Usar o GitHub como histórico de alterações do projeto.
4. Deixar o repositório público para avaliação acadêmica.
5. Restringir alterações diretas ao dono do repositório.
6. Revisar arquivos antes de cada envio para o GitHub.

## Gestão de usuários

O controle de usuários é feito pelo próprio GitHub. Nesta entrega, o repositório fica sob responsabilidade do aluno Raphael Victor Gomes Lima. Caso existam colaborações futuras, a recomendação é usar pull requests, revisão manual e permissões mínimas.

## Controle de acesso

O acesso de leitura ao site e ao repositório é público, pois a atividade pede compartilhamento e avaliação. O acesso de escrita deve permanecer restrito ao dono do repositório ou a colaboradores autorizados.

## Boas práticas recomendadas

- Ativar autenticação em duas etapas na conta do GitHub.
- Conferir permissões de colaboradores antes de aceitar alterações.
- Não armazenar arquivos pessoais que não façam parte do portfólio.
- Manter commits com mensagens claras.
- Revisar links externos periodicamente.
